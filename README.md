# sair-ic50-oracle

A deep learning model that predicts **pIC50** (binding affinity) from a protein sequence and a small molecule, trained on the [SAIR dataset](https://huggingface.co/datasets/SandboxAQ/SAIR). Serves as the reward function for a GFlowNet drug-discovery pipeline.

---

## Dataset and filtering

**Source:** SandboxAQ/SAIR on HuggingFace — 8.8 M protein-ligand interaction records with experimentally measured IC50 values and AlphaFold-docked poses.

**Filters applied (in order):**

| Filter | Rationale | Rows remaining |
|--------|-----------|---------------|
| `all_passed = True` | PoseBusters geometry checks — removes docked poses with invalid bond lengths, clashes, etc. | ~2.3 M |
| `assay = 'biochem'` | Biochemical assays only — cleaner IC50 signal than cell or homogenate assays | ~2.3 M |
| Protein annotation filter | See below | ~2.0 M |
| Deduplication `(sequence, SMILES)` | Keep one row per unique pair (highest ipTM wins) | **382,862** |

**Protein annotation filter.**  ESM2 can only embed the first 1,022 residues of a protein sequence. For proteins longer than 1,022 aa, the model would miss any binding-site residues that fall beyond that cutoff, producing a misleading embedding. The filter works as follows:
- For short proteins (≤ 1,022 aa): always kept.
- For long proteins (> 1,022 aa): kept only if UniProt binding-site annotations confirm that **all** annotated active-site residues fall within the first 1,022 positions.
- Long proteins with any binding-site residue beyond position 1,022, or with no UniProt annotation, are dropped.

This reduces the protein set from 5,149 to **3,563 proteins** and is documented in `data/long_protein_annotations.csv` (generated by `notebooks/01_data_exploration.ipynb`).

**pIC50 range:** 4.0 – 11.0 (100 µM to 10 pM). Most drug-like compounds of interest fall in the 7–9 range (10–1 nM).

**Splits** are protein-cluster-stratified using MMseqs2 at 80% sequence identity, preventing protein-family leakage between sets:

| Split | Proteins | Samples | % |
|-------|----------|---------|---|
| Train | 2,859 | 298,888 | 78 |
| Val | 354 | 41,269 | 11 |
| Test | 350 | 42,705 | 11 |

## Model architecture

The model fuses a **frozen protein embedding** with a **learned ligand graph embedding** and passes the concatenation through a regression MLP.

```
protein_emb [1280]  ──────────────────────────────┐
                                                    ├─ concat [1536] ──> Fusion MLP ──> pIC50 [1]
ligand_emb  [256]   ── GINEConv × 4 ── mean_pool ─┘
```

### Protein encoder

Precomputed ESM2-650M mean-pooled embeddings (`esm2_t33_650M_UR50D`). Each protein is embedded once offline (`scripts/precompute_esm.py`) and cached as a `[1280]` float32 tensor. The encoder is frozen during oracle training — no fine-tuning. An optional MLP projection layer (`project: true` in config) can reduce the dimension to 256 before fusion, but the default is the identity (pass-through).

### Ligand encoder

A 4-layer **GINEConv** (Graph Isomorphism Network with Edge features) operating on the 2D molecular graph constructed by RDKit.

**Atom features — 14-dimensional:**

| Index | Feature |
|-------|---------|
| 0–9 | One-hot symbol: C, N, O, S, F, P, Cl, Br, I, other |
| 10 | Formal charge |
| 11 | Total H count |
| 12 | Is aromatic (0/1) |
| 13 | Atomic mass / 100 |

**Bond features — 3-dimensional:**

| Index | Feature |
|-------|---------|
| 0 | Bond order / 3.0 (single=0.33, double=0.67, triple=1.0, aromatic=0.5) |
| 1 | Is conjugated (0/1) |
| 2 | Is in ring (0/1) |

**Layer structure:** Linear(14 → 256) input projection, then 4 × [GINEConv → BatchNorm1d → ReLU], Dropout(0.1), `global_mean_pool` readout. `global_mean_pool` is used instead of `global_add_pool` because it is size-invariant and more stable for regression.

### Fusion MLP

```
Linear(1536, 512) → ReLU → Dropout(0.1)
Linear(512, 256)  → ReLU → Dropout(0.1)
Linear(256, 1)
```

Output is an unbounded scalar in pIC50 space. No sigmoid or clamping — the loss and metrics operate directly in pIC50 space.

**Total parameters:** ~1.45 M (ligand encoder + fusion MLP only; protein encoder is frozen).

### GFlowNet compatibility invariants

`oracle/featurise.py` is the single source of truth shared with the sibling GFlowNet repo. Two constants must never be changed without updating both repos simultaneously:

- `ATOM_FEATURE_DIM = 14`
- `BOND_FEATURE_DIM = 3`

`mol_to_graph(mol)` accepts an RDKit `Mol` object directly and skips dummy atoms (atomic number 0, i.e. BRICS attachment points `*`), so it is safe to call on partial molecules during GFlowNet generation. Checkpoints include a `config` dict so `IC50Oracle.from_config(ckpt['config'])` reconstructs the model without needing to know the training hyperparameters.

## Loss function

The training objective combines a **Huber loss** with a **pairwise ranking loss**:

```
L = Huber(pred, target, δ=1.0)  +  0.1 × RankingLoss(pred, target, margin=0.5)
```

**Huber loss** is used instead of MSE because pIC50 labels from biochemical assays are noisy and contain outliers. Huber is quadratic for small errors and linear for large ones (controlled by δ=1.0), making it more robust without discarding outlier pairs entirely.

**Ranking loss** is a vectorised pairwise margin loss over same-protein pairs within each batch:

```
L_rank = mean over (i,j) where protein_i == protein_j:
    max(0,  margin − sign(target_i − target_j) × (pred_i − pred_j))
```

It penalises the model for inverting the ordering of two compounds against the same protein. This is critical for GFlowNet use: the oracle is used to rank candidate molecules, so ranking quality matters more than absolute pIC50 accuracy. Ranking weight 0.1 means the Huber loss dominates calibration while the ranking loss adds a soft ordering constraint.

## Training

| Hyperparameter | Value |
|----------------|-------|
| Optimizer | AdamW |
| Learning rate | 1e-4 |
| Weight decay | 1e-5 |
| Scheduler | CosineAnnealingLR (per step, η_min=1e-6) |
| Batch size | 256 |
| Max epochs | 50 |
| Early stopping patience | 10 epochs |
| Gradient clipping | 1.0 (max norm) |
| Mixed precision | configurable (`mixed_precision: true/false`) |

Early stopping monitors **val Spearman** (not val loss) because ranking quality is the deployment metric. The best checkpoint by val Spearman is saved to `checkpoints/best.pt`.

## Metrics

**Primary metric: Spearman correlation.** Measures monotonic rank agreement between predicted and true pIC50. This is the deployment metric — the GFlowNet needs the oracle to correctly order compounds, not to produce perfectly calibrated absolute values.

**Per-protein Spearman** is the most meaningful diagnostic: the average Spearman correlation computed separately within each protein's val/test compounds. A model can achieve a high global Spearman by exploiting between-protein pIC50 differences (some proteins simply have more potent ligands than others); per-protein Spearman measures whether the oracle correctly ranks compounds *against the same target*, which is what the GFlowNet actually needs.

**Additional metrics reported:** Pearson correlation, MAE, RMSE.

**Baselines** available directly from the SAIR parquet (no retraining needed):

| Baseline | Description |
|----------|-------------|
| `aevplig_score` | AEV-PLIG physics-based scoring |
| `onionnet_score` | OnionNet CNN-based scoring |
| `vina_score` | AutoDock Vina docking score |
| `vinardo_score` | Vinardo scoring function |

Baseline comparison is in `notebooks/02_model_evaluation.ipynb`. Docking scores are negated before computing Spearman (lower score = more potent, so negation aligns the direction with pIC50).

## Quickstart

```bash
# 1. Create environment
conda env create -f environment.yml
conda activate sair-oracle

# 2. Download data (requires HuggingFace access to SandboxAQ/SAIR)
python scripts/download_data.py --output-dir data/

# 3. Precompute ESM2 embeddings (~15-20 min on M4 Max)
python scripts/precompute_esm.py \
  --parquet      data/sair.parquet \
  --output-dir   data/esm_embeddings/ \
  --annotations  data/long_protein_annotations.csv \
  --model        esm2_t33_650M_UR50D \
  --batch-size   16 \
  --device       auto

# 4. Generate splits (requires mmseqs2)
python oracle/make_splits.py \
  --parquet      data/sair.parquet \
  --annotations  data/long_protein_annotations.csv \
  --output-dir   data/splits/

# 5. Train
nice -n 10 python scripts/train.py --config config/default.yaml

# 6. Evaluate — open notebooks/02_model_evaluation.ipynb
```
